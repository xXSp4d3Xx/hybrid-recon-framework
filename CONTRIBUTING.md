# Contributing

Thank you for wanting to contribute!

How to contribute
- Open an issue to discuss substantial changes before starting work.
- Fork the repository and create a feature branch.
- Keep PRs small and focused with clear commit messages.
- Include tests or repro steps where appropriate.

Style & quality
- Run `shellcheck` on shell scripts before submitting PRs (`make lint`).
- Prefer `set -euo pipefail` when safe and add comments for complex behavior.
- Provide documentation updates for changes that affect usage.

License
- By contributing, you agree to license your contributions under the project's MIT license.
